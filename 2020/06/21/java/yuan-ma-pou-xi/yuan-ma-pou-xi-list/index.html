List相关类源码剖析

一. ArrayList

1. 总结

ArrayList是有序的，底层使用的是数组结构(可扩容)，查询快，增删慢。
ArrayList线程不安全。
默认容量为0，第一次添加元素，扩容为10，之后数组元素数量超过数组长度，则扩容为之前的1.5倍。

2. 源码解析

public class ArrayList<E> extends AbstractList<E>
        implements List<E>, RandomAccess, Cloneable, java.io.Serializable{}

2.1 Field

//序列化
private static final long serialVersionUID = 8683452581122892189L;
//默认容量为10
private static final int DEFAULT_CAPACITY = 10;
//空数组
private static final Object[] EMPTY_ELEMENTDATA = {};
//使用无参构造函数时，使用该数组
private static final Object[] DEFAULTCAPACITY_EMPTY_ELEMENTDATA = {};
//数据对象存放地方，当前对象不参与序列化
transient Object[] elementData; 
//数组元素长度
private int size;
//要分配的数组的最大大小
private static final int MAX_ARRAY_SIZE = Integer.MAX_VALUE - 8;

2.2 Constructor

//有参构造 确定数组的大小
public ArrayList(int initialCapacity) {
    if (initialCapacity > 0) {
        this.elementData = new Object[initialCapacity];
    } else if (initialCapacity == 0) {
        this.elementData = EMPTY_ELEMENTDATA;
    } else {
        throw new IllegalArgumentException("Illegal Capacity: "+
                                           initialCapacity);
    }
}

//无参构造 数组容量默认为0
public ArrayList() {
    this.elementData = DEFAULTCAPACITY_EMPTY_ELEMENTDATA;
}

//Collection是一个接口 将该接口的某个实现类转化为ArrayList，当然该实现类中的元素必须是ArrayList元素的子类
public ArrayList(Collection<? extends E> c) {
    elementData = c.toArray();
    if ((size = elementData.length) != 0) {
        // c.toArray might (incorrectly) not return Object[] (see 6260652)
        if (elementData.getClass() != Object[].class)
            elementData = Arrays.copyOf(elementData, size, Object[].class);
    } else {
        // replace with empty array.
        this.elementData = EMPTY_ELEMENTDATA;
    }
}

2.3 Method(核心方法)

//向ArrayList中添加一个元素
public boolean add(E e) {
    ensureCapacityInternal(size + 1);  // Increments modCount!!
    elementData[size++] = e;
    return true;
}

//得到最小扩容量
private void ensureCapacityInternal(int minCapacity) {
    ensureExplicitCapacity(calculateCapacity(elementData, minCapacity));
}

//计算容量
private static int calculateCapacity(Object[] elementData, int minCapacity) {
    //无参构造函数 返回此语句
    if (elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA) {
        return Math.max(DEFAULT_CAPACITY, minCapacity);
    }
    //有参构造函数 返回此语句
    return minCapacity;
}

//判断是否需要扩容
private void ensureExplicitCapacity(int minCapacity) {
     //修改次数,防止多线程环境下结构破坏
     modCount++;
     // overflow-conscious code
     //如果最小空间比elementData空间大，则需要扩容
     if (minCapacity - elementData.length > 0)
         grow(minCapacity);
}

//ArrayList动态扩容的核心方法
private void grow(int minCapacity) {
    // overflow-conscious code
    int oldCapacity = elementData.length;
    //扩容至原来的1.5倍
    int newCapacity = oldCapacity + (oldCapacity >> 1);
    //判断新数组的容量够不够，够了就直接使用这个长度创建新数组
    //不够就将数组长度设置为需要的长度
    if (newCapacity - minCapacity < 0)
        newCapacity = minCapacity;
    //判断有没有超过最大限制
    if (newCapacity - MAX_ARRAY_SIZE > 0)
        newCapacity = hugeCapacity(minCapacity);
    // minCapacity is usually close to size, so this is a win:
    elementData = Arrays.copyOf(elementData, newCapacity);
}

//序列化
private void writeObject(java.io.ObjectOutputStream s)
    throws java.io.IOException{
    int expectedModCount = modCount;
    s.defaultWriteObject();
    s.writeInt(size);
    //因为ArrayList的一些空间可能没有使用，采用这种方式来节约空间和减少序列化的时间
    for (int i=0; i<size; i++) {
        s.writeObject(elementData[i]);
    }
    if (modCount != expectedModCount) {
        throw new ConcurrentModificationException();
    }
}

2.4 Inner Class

//实现普通迭代器Iterator
private class Itr implements Iterator<E> {}
//实现迭代器ListIterator，比Iterator多了几个方法
private class ListItr extends Itr implements ListIterator<E> {}
//子列表，用于从列表中截取一段
private class SubList extends AbstractList<E> implements RandomAccess {}
////实现Java8新添加的接口Spliterator，可分割的迭代器，主要用于并发
static final class ArrayListSpliterator<E> implements Spliterator<E> {}

二. LinkedList

1. 总结

LinkedList底层是一个双链表，是一个直线型的链表结构。
LinkedList查询慢，插入删除快，线程不安全。
由于LinkedList是双向链表，没有初始化大小，没有扩容机制。
推荐采用迭代器遍历，速度远大于for循环遍历。

2. 源码

public class LinkedList<E>
    extends AbstractSequentialList<E>
    implements List<E>, Deque<E>, Cloneable, java.io.Serializable{}

2.1 Field

//LinkedList的长度
transient int size = 0;
//头结点
transient Node<E> first;
//尾结点
transient Node<E> last;
//序列化ID
private static final long serialVersionUID = 876323262645176354L;

2.2 Constructor

//无参构造
public LinkedList() {
}

//该接口的某个实现类转化为LinkedList
public LinkedList(Collection<? extends E> c) {
    this();
    addAll(c);
}

2.3 Method(核心方法)

//将集合c添加到链接到指定索引的后面
public boolean addAll(int index, Collection<? extends E> c) {
    //几乎所有的涉及到在指定位置添加或者删除或修改操作都需要判断传进来的参数是否合法。
    checkPositionIndex(index);
    Object[] a = c.toArray();
    int numNew = a.length;
    if (numNew == 0)
        return false;
    Node<E> pred, succ;
    //新添加的元素的位置位于LinkedList最后一个元素的后面。
    if (index == size) {
        succ = null;
        pred = last;
    //新添加的元素的位置位于LinkedList中。
    } else {
        succ = node(index);
        pred = succ.prev;
    }
    //遍历数组，添加节点
    for (Object o : a) {
        @SuppressWarnings("unchecked") E e = (E) o;
        Node<E> newNode = new Node<>(pred, e, null);
        if (pred == null)
            first = newNode;
        else
            pred.next = newNode;
        pred = newNode;
    }
    if (succ == null) {
        last = pred;
    } else {
        pred.next = succ;
        succ.prev = pred;
    }
    size += numNew;
    modCount++;
    return true;
}

//返回索引上的节点
//LinkedList还对整个做了优化，不是盲目地直接从头进行遍历，
//而是先比较一下index更靠近链表（LinkedList）的头节点还是尾节点。
//然后进行遍历，获取相应的节点。
Node<E> node(int index) {
    // assert isElementIndex(index);
    if (index < (size >> 1)) {
        Node<E> x = first;
        for (int i = 0; i < index; i++)
            x = x.next;
        return x;
    } else {
        Node<E> x = last;
        for (int i = size - 1; i > index; i--)
            x = x.prev;
        return x;
    }
}

//把参数中的元素作为链表的第一个元素
private void linkFirst(E e) {
    final Node<E> f = first;
    final Node<E> newNode = new Node<>(null, e, f);
    first = newNode;
    //此时链表为空
    if (f == null)
        last = newNode;
    else
        f.prev = newNode;
    size++;
    modCount++;
}

//把参数中的元素作为链表的最后一个元素
void linkLast(E e) {
    final Node<E> l = last;
    final Node<E> newNode = new Node<>(l, e, null);
    last = newNode;
    //此时链表为空
    if (l == null)
        first = newNode;
    else
        l.next = newNode;
    size++;
    modCount++;
}

//在非空节点succ之前插入元素e
void linkBefore(E e, Node<E> succ) {
    // assert succ != null;
    final Node<E> pred = succ.prev;
    final Node<E> newNode = new Node<>(pred, e, succ);
    succ.prev = newNode;
    //
    if (pred == null)
        first = newNode;
    else
        pred.next = newNode;
    size++;
    modCount++;
}

//删除first节点，并且规定first节点不能为空
private E unlinkFirst(Node<E> f) {
    // assert f == first && f != null;
    //保存相应的值
    final E element = f.item;
    final Node<E> next = f.next;
    f.item = null;
    f.next = null; // help GC 让GC回收
    first = next;
    if (next == null)
        last = null;
    else
        next.prev = null;
    size--;
    modCount++;
    return element;
}

//删除first节点并返回
public E removeFirst() {
    final Node<E> f = first;
    if (f == null)
        throw new NoSuchElementException();
    return unlinkFirst(f);
}

//从LinkedList中删除指定元素。
//(且只删除第一次出现的指定的元素)(如果指定的元素在集合中不存在，则返回false，否则返回true)
public boolean remove(Object o) {
    if (o == null) {
        for (Node<E> x = first; x != null; x = x.next) {
            if (x.item == null) {
                unlink(x);
                return true;
            }
        }
    } else {
        for (Node<E> x = first; x != null; x = x.next) {
            if (o.equals(x.item)) {
                unlink(x);
                return true;
            }
        }
    }
    return false;
}

//判断index是否是元素的索引
private boolean isElementIndex(int index) {
    return index >= 0 && index < size;
}

//LinkedList的这个方法用于判断当我们新添加元素的时候，传进来的index是否合法，
//而且我们新添加的元素可能在LinkedList最后一个元素的后面，所以这里允许index<=size。
private boolean isPositionIndex(int index) {
    return index >= 0 && index <= size;
}

    Node<E> prev;
    Node(Node<E> prev, E element, Node<E> next) {
        this.item = element;

三. Vector

1. 总结

Vector和ArrayList底层数据结构一样是数组，Vector可以设置元素的个数，ArrayList不可以。Vector是线程安全的。
Vector相比ArrayList多了一个增长系数属性。
Vector默认容量为10、增长系数为0，默认扩容为之前的2倍。

2. 源码

public class Vector<E>
    extends AbstractList<E>
    implements List<E>, RandomAccess, Cloneable, java.io.Serializable{}

2.1 Field

//数组元素存储缓冲区
protected Object[] elementData;
//数组元素个数
protected int elementCount;
//容量增长系数：默认是增加一倍
protected int capacityIncrement;
//序列化ID
private static final long serialVersionUID = -2767605614048989439L;
//要分配的数组的最大大小
private static final int MAX_ARRAY_SIZE = Integer.MAX_VALUE - 8;

2.2 Constructor

//指定数组初始化容量大小和增长系数
public Vector(int initialCapacity, int capacityIncrement) {
    super();
    if (initialCapacity < 0)
        throw new IllegalArgumentException("Illegal Capacity: "+
                                           initialCapacity);
    this.elementData = new Object[initialCapacity];
    this.capacityIncrement = capacityIncrement;
}

//指定Vector初始化容量大小，默认增长系数为0
public Vector(int initialCapacity) {
    this(initialCapacity, 0);
}

//无参构造，初始容量为10，增长系数为0
public Vector() {
    this(10);
}

//构造一个包含指定集合C中的元素的数组
public Vector(Collection<? extends E> c) {
    elementData = c.toArray();
    elementCount = elementData.length;
    // c.toArray might (incorrectly) not return Object[] (see 6260652)
    if (elementData.getClass() != Object[].class)
        elementData = Arrays.copyOf(elementData, elementCount, Object[].class);
}

2.3 Method(核心方法)

//加锁的元素添加方法
public synchronized boolean add(E e) {
    modCount++;
    ensureCapacityHelper(elementCount + 1);
    elementData[elementCount++] = e;
    return true;
}

//判断是否需要扩容
private void ensureCapacityHelper(int minCapacity) {
    // overflow-conscious code
    if (minCapacity - elementData.length > 0)
        grow(minCapacity);
}

//扩容的核心方法
private void grow(int minCapacity) {
    // overflow-conscious code
    int oldCapacity = elementData.length;
    //如果有给capacityIncrement设置增长系数的话
    //就加上该系数值来扩容，否则将原先的数组容量变为2*oldCapacity
    int newCapacity = oldCapacity + ((capacityIncrement > 0) ?
                                     capacityIncrement : oldCapacity);
    if (newCapacity - minCapacity < 0)
        newCapacity = minCapacity;
    if (newCapacity - MAX_ARRAY_SIZE > 0)
        newCapacity = hugeCapacity(minCapacity);
    elementData = Arrays.copyOf(elementData, newCapacity);
}

//设置Vector的元素个数,同步的
public synchronized void setSize(int newSize) {
    modCount++;
    if (newSize > elementCount) {
        ensureCapacityHelper(newSize);
    } else {
        for (int i = newSize ; i < elementCount ; i++) {
            elementData[i] = null;
        }
    }
    elementCount = newSize;
}

2.4 Inner Class

private class Itr implements Iterator<E> {}
//实现ListIterator迭代器
final class ListItr extends Itr implements ListIterator<E> {}
//实现分割迭代器Spliterator
static final class VectorSpliterator<E> implements Spliterator<E> {}

四. CopyOnWriteArrayList

1. 总结

该类在并发包java.util.concurrent下，显然是线程安全的。
CopyOnWriteArrayList底层数据结构是数组，默认容量为0，因为采用读写分离，所以是添加1个元素数组长度则加1。
对比： CopyOnWriteArrayList相比于Vector，并发性提高了很多。
优点：写时复制容器(适合多读少写情况)，当某个线程在添加元素时，不会影响另外一个线程读取该列表，因为在添加元素时会先复制一份数组，这样读和写的容器相当于分离(写加锁，读不加锁)。
缺点：由于写操作需要复制整个容器，会造成内存占有大。
缺点：由于写操作是在新数组上进行，读操作不能立即可见，有可能会产生脏读。

2. 源码

package java.util.concurrent;
public class CopyOnWriteArrayList<E>
    implements List<E>, RandomAccess, Cloneable, java.io.Serializable {}

2.1 Field

//可重入锁
final transient ReentrantLock lock = new ReentrantLock();
//数据对象存放地方
private transient volatile Object[] array;

2.2 Constructor

//默认构造函数 数组长度为0
public CopyOnWriteArrayList() {
    setArray(new Object[0]);
}

//构造一个包含指定集合C中的元素的数组
public CopyOnWriteArrayList(Collection<? extends E> c) {
    Object[] elements;
    if (c.getClass() == CopyOnWriteArrayList.class)
        elements = ((CopyOnWriteArrayList<?>)c).getArray();
    else {
        elements = c.toArray();
        // c.toArray might (incorrectly) not return Object[] (see 6260652)
        if (elements.getClass() != Object[].class)
            elements = Arrays.copyOf(elements, elements.length, Object[].class);
    }
    setArray(elements);
}

//将传入的数组赋给array数组
public CopyOnWriteArrayList(E[] toCopyIn) {
    setArray(Arrays.copyOf(toCopyIn, toCopyIn.length, Object[].class));
}

2.3 Method

//加锁添加数据 先复制一份数组 再对新数组操作 
public boolean add(E e) {
    final ReentrantLock lock = this.lock;
    lock.lock();
    try {
        Object[] elements = getArray();
        int len = elements.length;
        Object[] newElements = Arrays.copyOf(elements, len + 1);
        newElements[len] = e;
        setArray(newElements);
        return true;
    } finally {
        lock.unlock();
    }
}

//加锁删除数据 先复制一份数组 再对新数组操作 返回删除的值
public E remove(int index) {
    final ReentrantLock lock = this.lock;
    lock.lock();
    try {
        Object[] elements = getArray();
        int len = elements.length;
        E oldValue = get(elements, index);
        int numMoved = len - index - 1;
        //删除的元素是数组的最后一个元素
        if (numMoved == 0)
            setArray(Arrays.copyOf(elements, len - 1));
        else {
            Object[] newElements = new Object[len - 1];
            System.arraycopy(elements, 0, newElements, 0, index);
            System.arraycopy(elements, index + 1, newElements, index,
                             numMoved);
            setArray(newElements);
        }
        return oldValue;
    } finally {
        lock.unlock();
    }
}

//读操作没有加锁
private E get(Object[] a, int index) {
    return (E) a[index];
}

//数组中没有此值时才添加
private boolean addIfAbsent(E e, Object[] snapshot) {
    final ReentrantLock lock = this.lock;
    lock.lock();
    try {
        Object[] current = getArray();
        int len = current.length;
        if (snapshot != current) {
            // Optimize for lost race to another addXXX operation
            int common = Math.min(snapshot.length, len);
            for (int i = 0; i < common; i++)
                if (current[i] != snapshot[i] && eq(e, current[i]))
                    return false;
            if (indexOf(e, current, common, len) >= 0)
                    return false;
        }
        Object[] newElements = Arrays.copyOf(current, len + 1);
        newElements[len] = e;
        setArray(newElements);
        return true;
    } finally {
        lock.unlock();
    }
}

五. 有关List的面试题

（1） 你知道的List都有哪些？

  • ArrayList、Vector、LinkedList、CopyOnWriteArrayList。

（2）List是有序的吗？

  • List是有序的。

（3）List和Vector有什么区别？

  • Vector是List接口下线程安全的集合。

（4）ArrayList和LinkedList的底层数据结构是什么？

  • ArrayList使用的是数组结构，LinkedList使用的是链表结构。

（5）ArrayList默认大小是多少，是如何扩容的？

  • Jdk1.7之前ArrayList默认大小是10，JDK1.7之后是0，JDK差异，每次约按1.5倍扩容。

（6）ArrayList和LinkedList的区别？分别用在什么场景？

  • ArrayList和LinkedList数据结构不一样，前者用在查询较多的场合，后者适用于插入较多的场合。

（7）List是线程安全的吗？如果要线程安全要怎么做？

  • List中的Vector、CopyOnWriteArrayList才是线程安全的，其他要实现线程安全使用工具类Collections.synchronizedList(new ArrayList())方法。

（8）怎么给List排序？

  • 使用List自身的sort方法，或者使用Collections.sort(list)方法;

（9）Arrays.asList方法后的List可以扩容吗？

  • Arrays.asList使用的是final数组，返回的ArrayList是Arrays的一个内部类，该内部类中没有add、remove方法，不支持扩容。

（10）List和Array之间如何互相转换？

  • List转Array使用toArray方法，Array转List使用Arrays.asList(array)方法，由于它是固定的，不固定的可以使用new ArrayList(Arrays.asList(array))。
